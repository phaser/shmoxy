## Description

Re-apply body syntax highlighting when navigating requests via arrow keys in detail dialog (#335)

## Notes

Fixes #335.

### Problem
In `InspectionDetail.razor`, body syntax highlighting (`HighlightCode()` → `syntaxHighlight.highlightAll`)
was only invoked inside `OnAfterRenderAsync` when `firstRender == true`. Prev/next arrow navigation
(`NavigateToAdjacentRow` → `OnNavigate`) updates `selectedRow` in the parent `Inspection.razor`, which
reuses the same `InspectionDetail` instance (no `@key`) and merely swaps the `Row` parameter. The
follow-up render has `firstRender == false`, so highlighting was never re-applied to the new body.

### Fix
Track the last-highlighted row (`highlightedRow`) and re-run `HighlightCode()` from `OnAfterRenderAsync`
whenever the displayed `Row` reference changes. Reference equality is used so the change is independent
of the `Id` type and does not double-fire after tab/JWT toggles (which already call `HighlightCode()`
explicitly and keep the same `Row`). The focus-on-first-render behaviour is preserved.

This was chosen over adding `@key="selectedRow.Id"` to avoid tearing down and recreating the dialog
(and resetting tab/JWT-expansion state) on every navigation.
