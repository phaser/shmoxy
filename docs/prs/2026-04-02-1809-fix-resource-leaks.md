Fix resource leaks and code quality issues found during code review.

## Changes

### Resource leaks (IDisposable)

1. **InspectionController.GetRemoteStream** -- `HttpClientHandler`, `HttpClient`, `LoggerFactory`, and `ProxyIpcClient` were created but never disposed. Added `using` declarations so they are disposed when the async iterator completes.

2. **RemoteProxyHealthMonitor.CheckHealthWithClientAsync** -- Same pattern: `HttpClientHandler`, `HttpClient`, `LoggerFactory`, and `ProxyIpcClient` leaked on every health check. Added `using` declarations.

3. **RemoteProxyRegistry.TestConnectivityAsync** -- Same pattern: `HttpClientHandler`, `HttpClient`, and `ProxyIpcClient` leaked on every connectivity test. Added `using` declarations.

4. **ProxyControlApi /ipc/inspect/stream** -- `CancellationTokenSource` was created but never disposed or cancelled. Replaced with `HttpContext.RequestAborted` token, which properly signals when the client disconnects.

### Code correctness

5. **ProxyServer.RelayAsync** -- `catch (Exception) when (ct.IsCancellationRequested || true)` had a misleading exception filter. The `|| true` made the guard always evaluate to true, rendering the `ct.IsCancellationRequested` check dead code. Simplified to `catch (Exception)` since the intent is to catch all exceptions during relay (IO errors, cancellation, connection resets are all expected).

6. **IpcApiTests.Hooks_Endpoint_ListsInspectionHook** -- Test asserted `enabled == false` but `InspectionHook` defaults to `enabled = true`. The test only passed by coincidence when `Inspection_Enable_Disable_TogglesHook` happened to run first and leave the hook disabled. Fixed to assert the property exists without depending on execution order.

### Performance

7. **ApiClient.StreamInspectionEventsAsync** -- Allocated a new `JsonSerializerOptions` on every SSE line, causing repeated reflection/metadata work. Extracted to a static field shared across all calls.
