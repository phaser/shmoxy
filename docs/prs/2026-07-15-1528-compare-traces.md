## Description

Compare two saved traces side by side from the Saved Traces page — a diff view
that surfaces regressions between two captures of the same endpoint (different
runs, environments, auth states, versions).

## Scope

- **Diff engine (`services/DiffService.cs`, client-side, no new dependency):**
  - `DiffLines` — LCS line diff with common prefix/suffix trimming; consecutive
    removed+added runs paired into `Changed` rows.
  - `DiffNamedValues` — aligns headers / query params by name (added / removed /
    changed / unchanged), collapsing repeated names.
  - `DiffUrls` — structural URL diff (origin+path as base string, query params
    aligned by name, values URL-unescaped).
  - `DiffBody` — pretty-prints via existing `PayloadFormatter` then line-diffs;
    binary (base64) and oversized bodies (`MaxDiffableBodyChars` / `MaxDiffableLines`)
    degrade to equality + sizes only.
  - Diff models in `models/` (one type per file): `DiffKind`, `NamedValueDiff`,
    `LineDiffRow`, `UrlDiff`, `BodyDiff`, `BodyDiffKind`.
- **UI:**
  - `components/TraceComparison.razor` overlay: side-labelled two-column view with
    URL (structural + query table), Response (status/duration/size/timing with
    numeric deltas), request+response header diff tables (unchanged rows collapsed
    behind a "show N unchanged" toggle), and side-by-side body line diffs. Swap-sides
    control; Esc / × to close.
  - `pages/SavedTraces.razor`: checkbox column, `Compare (n/2)` toolbar button
    enabled only when exactly two **non-WebSocket** traces are selected (disabled
    with a hint otherwise); selection is transient. Both full traces fetched via the
    existing `GET /api/saved-traces/{id}` — no new API endpoints.

## Design decisions

- Older capture (by original timestamp) shown left, newer right; swap toggles.
- WebSocket traces excluded from v1 per the issue (Compare disabled with a hint).
- In-repo LCS chosen over a NuGet diff library — no license-verification/dependency
  overhead, and the alignment logic is small.
